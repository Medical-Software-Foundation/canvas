## Key issues caught and fixed during this engagement

These were not caught by unit tests (the handler tests deliberately mock the ORM call) and only surfaced under UAT — worth noting for future sessions:

1. **`NoteStates.LKD` AttributeError.** Initial implementation used the wire values as if they were enum member names. The Django `TextChoices` members are `LOCKED` / `RELOCKED` / `SIGNED`; values are `"LKD"` / `"RLK"` / `"SGN"`. Fixed in v0.1.5.
2. **Missing `SIGNED` in the locked-state filter.** Locking a note auto-progresses `LKD → SGN`, so `CurrentNoteStateEvent` settles on `SGN`. The original spec under-specified this. Fixed in v0.1.5.
3. **`note__provider_id` returns Staff PK (`dbid`), not the human-facing string `id`.** `Staff` has a custom `id = CharField(db_column="key")` but does NOT mark it as the primary key — Staff's PK is `dbid` (BigAutoField). The FK column references `dbid`, but the rest of the dashboard operates on the string `id`. Fixed in v0.1.9 by using `note__provider__id` (relation traversal).
4. **RestrictedPython sandbox rejections.** `provider_counts[pid]["count"] += 1` (augmented assignment on subscript), `from django.conf import settings`, `staff: object` (object not in builtins), and `__import__(...)` were all rejected at module-load time. Each fixed in turn (v0.1.1 / v0.1.2 / v0.1.3).

The recurring shape: bugs in the ORM-wrapper layer (`_fetch_locked_state_events`, `_fetch_provider_counts`) are not caught by unit tests because the tests mock that layer. Future work should rely on UAT to catch these, or invest in integration tests against a real Canvas DB.

## Outstanding considerations (not blockers)

- **Times are formatted in UTC.** The plugin sandbox forbids `from django.conf import settings`, so the original "instance timezone" intent had to be replaced with a `_SettingsShim` defaulting to UTC. If localized display times are later required, swap the shim for a plugin secret read inside the handler (the `_window`/`_format_dos` helpers already accept tz indirectly via the shim).
- **`.only()` micro-optimization** — LOW finding from the DB performance review. Skipping `Note.body` / `Note.related_data` JSON columns would reduce per-row payload but provides modest gain; left for later if profiling indicates it.
