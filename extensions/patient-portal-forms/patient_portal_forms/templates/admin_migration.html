Patient Portal Forms — Metadata Migration

This plugin previously stored assigned questionnaires in PatientMetadata under the key portal_forms. Two steps move that data into the new QuestionnaireAssignment table (namespace canvas__patient_portal_forms) and then clean up the legacy rows. Both steps are idempotent.

Step 1 — Copy legacy data into the new table

Walks every portal_forms metadata row and creates equivalent QuestionnaireAssignment rows. Safe to run more than once — patients already migrated will be skipped.

Step 2 — Clear the legacy metadata values

Overwrites the value of every portal_forms PatientMetadata row with an empty string. The row itself remains — the SDK only supports upsert on PatientMetadata, not delete — but its data is neutralized so nothing reads it again. Only run this after Step 1 has succeeded and you have spot-checked the new assignments.
